# Incident Track Frontend Study Guide Index

This folder is a study set for the frontend. The goal is not just to "know what file does what", but to understand the project well enough to:

- explain it clearly in an interview
- modify existing features without breaking flow
- recreate major parts from scratch
- answer React questions using real examples from this codebase

These notes are intentionally written more like personal technical study notes than a short project summary.

## Recommended reading order

1. `README-01-Setup-And-Bootstrap.md`
2. `README-02-Project-Structure-And-Folders.md`
3. `README-03-Entry-Routing-And-Authentication.md`
4. `README-04-Shared-UI-Context-And-Data-Layer.md`
5. `README-05-Feature-Implementation-Module-By-Module.md`
6. `README-06-React-Concepts-Hooks-Props-And-Interview-Prep.md`
7. `README-07-React-Deep-Dive-Personal-Notes.md`
8. `README-08-Rebuild-From-Scratch-Roadmap.md`

## How to study these files

- First pass: read all files in order without opening code constantly.
- Second pass: keep the related source files open and map each note to the real implementation.
- Third pass: use Phase 6 and Phase 7 as interview prep notes.
- Fourth pass: use Phase 8 and try mentally rebuilding the project layer by layer.

## Best files to keep open while reading code

- Keep Phase 3 open when reading routing and auth files.
- Keep Phase 4 open when reading layout, shared components, and contexts.
- Keep Phase 5 open when reading `pages` and `features`.
- Keep Phase 6 and 7 open when preparing for React interviews.

## Outcome expected from this study set

After finishing these notes properly, you should be able to explain:

- how the frontend starts
- how routes are protected
- how data flows from API layer to page UI
- how global state is handled
- why specific React hooks are used in specific files
- how to add a new feature in the same style as the existing code
